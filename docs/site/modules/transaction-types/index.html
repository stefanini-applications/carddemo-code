🧾 Transaction Types

Guide for stories that expand the DB2-backed reference data and keep VSAM in sync.

📋 Overview

Purpose: Extend the admin menu with CRUD operations over transaction type metadata, backed by DB2 tables and mirrored into VSAM so runtime processing stays performant.

Business Context: Allows admins to edit transaction descriptions, categories, and metadata in a relational store while still feeding VSAM-based workflows.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Metadata edit: As an admin, I want to add a new transaction type via `CTTU` so new merchant categories appear in reports.
  • Sync flow: As a data steward, I want `TRANEXTR` to export DB2 rows to VSAM nightly so transaction processing uses fresh data.
  • Bulk update: As a release engineer, I want `MNTTRDB2` to update descriptions in batch when regulatory text changes.

⚡ Development Acceleration Factors

  • SQL declarations: The `app/app-transaction-type-db2/dcl` files already declare host variables and SQLCA handling, so new statements follow the same scaffolding.
  • DB2 schema: `CREADB21` establishes `TRANSACTION_TYPE` and `TRANSACTION_TYPE_CATEGORY`, so follow their column definitions when extending metadata.
  • Extraction job: `TRANEXTR` uses `DSNTIAUL` to create VSAM-friendly files, so new fields just need updates in the extract routine and target copybook.

📊 Complexity Guidelines

  • Simple (1-2 pts): Add a new description field to `CTTU` and update the associated BMS map.
  • Medium (3-5 pts): Introduce a new `TRANSACTION_TYPE_CATEGORY` link, update `CTLI` to expose it, and add SQL `JOIN`s with SQLCA handling.
  • Complex (5-8 pts): Extend `TRANEXTR` and downstream VSAM readers so that newly captured metadata becomes available to `POSTTRAN` and reporting.

🔧 Technical Foundation

🏗️ Key Components

  • CICS transactions: `CTTU` (`COTRTUP`) for add/edit and `CTLI` (`COTRTLI`) for list/update/delete.
  • DB2 tables: `TRANSACTION_TYPE` (code + description) and `TRANSACTION_TYPE_CATEGORY` (category metadata with foreign key to type).
  • JCL jobs: `CREADB21` seeds tables, `TRANEXTR` exports data, `MNTTRDB2` runs bulk updates (`COBTUPDT`).
  • Copybooks: SQL host variables and list structures live under `app/app-transaction-type-db2/cpy` and `dcl`.

🔗 Dependencies

  • DB2 subsystem with plan `DB201PLN` registered in `app/app-transaction-type-db2/csd`.
  • VSAM layers (`TRANSACT.VSAM`) that consume the extracted metadata.
  • Admin menu option numbers (5/6) enabled when this module is installed.

🌐 Public Interfaces

CTTU  (COTRTUP) - Add/edit transaction type (static SQL with SQLCA)
CTLI  (COTRTLI) - List/update/delete with forward/backward cursors
CREADB21 - Batch job to create DB2 tables
TRANEXTR - Extract DB2 rows to VSAM files for runtime use
MNTTRDB2 - Batch maintenance job for large updates

✅ Acceptance Criteria Examples

  • Inserts via `CTTU` must write to `TRANSACTION_TYPE` and commit if SQLCA shows success.
  • `CTLI` must refuse deletes with dependent rows and display the reason via the SQLCA message area.
  • `TRANEXTR` must produce VSAM files used by `POSTTRAN`, and the job should succeed with return code zero.
