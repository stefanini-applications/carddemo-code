🏦 Account Management

User Story Development Guide — Account View & Update

← Back to Documentation Index

Module ID: account-management

Purpose: Enables regular users to view and update credit account information: balance, credit limits, status, open/expiration dates, and account group ID.

Tags: CICS VSAM

🎯 User Story Development Guide

📝 US Templates

  • View Account: As a card holder, I want to view my account details so that I can see my current balance, credit limit, and account status.
  • Update Account: As a card holder, I want to update my account ZIP code so that my statement is mailed to the correct address.
  • Credit Limit Display: As a card holder, I want to see my available credit so that I can plan my purchases.
  • Account Status: As a card holder, I want to view my account active status so that I know whether my account can accept new charges.
  • Account Group: As an admin, I want to assign an account to a disclosure group so that the correct interest rates are applied.

⚡ Development Acceleration Factors

  • CVACT01Y copybook: Account record layout is shared — used in view, update, and all batch programs
  • CXACAIX (AIX): Alternate index on ACCTDAT enables lookup by card number; reused in bill payment
  • COCOM01Y COMMAREA: Account number passed from prior screen; no re-entry needed
  • CSDAT01Y / CSMSG01Y: Shared date and message handling utilities

📊 Complexity Guidelines

1–2 pts
Simple
  • Add display of cash credit limit to view screen
  • Add read-only field for reissue date
3–5 pts
Medium
  • Add validation for credit limit update (must be ≥ current balance)
  • Add account status toggle with confirmation
5–8 pts
Complex
  • Add credit limit increase request workflow (approval required)
  • Implement multi-account display with pagination

🔧 Technical Foundation

🏗️ Key Components

  • COACTVWC — Account view; reads ACCTDAT via account ID from COMMAREA; resolves customer via CXACAIX
  • COACTUPC — Account update; reads for update (CICS READ UPDATE) then rewrites record
  • COACTVW / COACTUP (BMS maps) — View and update 3270 screen layouts
  • CVACT01Y (copybook) — Account record structure (300 bytes)
  • CBACT01C–CBACT04C (batch) — Account file sequential reads and reporting utilities

🔗 Dependencies

  • ACCTDAT (VSAM KSDS): Account master file, keyed on ACCT-ID (11-digit)
  • CXACAIX (VSAM AIX): Alternate index on ACCTDAT for card number lookups
  • CARDXREF (VSAM KSDS): Maps card number → customer ID → account ID
  • Authentication module: Account ID is passed via COMMAREA from signon

🌐 CICS Transactions

CAVW  → COACTVWC  — Account view (read-only)
CAUP  → COACTUPC  — Account update (read/write)

📋 Account Data Model (CVACT01Y, 300 bytes)

ACCOUNT-RECORD:
  ACCT-ID               PIC 9(11)       — Account ID (KSDS key)
  ACCT-ACTIVE-STATUS    PIC X(01)       — 'Y'=Active, 'N'=Inactive
  ACCT-CURR-BAL         PIC S9(10)V99   — Current balance
  ACCT-CREDIT-LIMIT     PIC S9(10)V99   — Total credit limit
  ACCT-CASH-CREDIT-LIMIT PIC S9(10)V99  — Cash advance limit
  ACCT-OPEN-DATE        PIC X(10)       — YYYY-MM-DD
  ACCT-EXPIRAION-DATE   PIC X(10)       — YYYY-MM-DD
  ACCT-REISSUE-DATE     PIC X(10)       — YYYY-MM-DD
  ACCT-CURR-CYC-CREDIT  PIC S9(10)V99   — Cycle credits
  ACCT-CURR-CYC-DEBIT   PIC S9(10)V99   — Cycle debits
  ACCT-ADDR-ZIP         PIC X(10)       — ZIP code
  ACCT-GROUP-ID         PIC X(10)       — Disclosure group ID

📋 Business Rules

  • BR-ACCT-01: Account status 'Y' = active; only active accounts accept new transactions
  • BR-ACCT-02: Current balance must not exceed the credit limit
  • BR-ACCT-03: Cash credit limit is separate from purchase credit limit
  • BR-ACCT-04: ACCT-GROUP-ID links to DISCGRP VSAM file for interest rate determination
  • BR-ACCT-05: Available credit = ACCT-CREDIT-LIMIT - ACCT-CURR-BAL

✅ Acceptance Criteria Examples

  • View Account: Given a valid account ID, when the user accesses CAVW, then all account fields are displayed including balance, credit limit, open date, and group ID
  • Update ZIP: Given the account update screen (CAUP), when a valid ZIP code is entered and confirmed, then the ZIP is updated in ACCTDAT
  • Validation: Given an update attempt to set credit limit below current balance, when the user submits, then an error message is displayed and the update is rejected
  • Performance: Account view must display within 3 seconds of navigating to CAVW
