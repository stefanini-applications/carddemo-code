🏦 Account Management

User Story Development Guide · CardDemo System

📋 Module Overview

Purpose: View and update credit card accounts including balances, credit limits, and account status.

Business Context: CSRs and credit officers need to look up account details to answer customer inquiries and make adjustments to credit limits or account status. Account data is the central entity linking customers, cards, and transactions.

Transaction IDs: CAVW (View) CAUP (Update)

Key Programs: COACTVWC COACTUPC

VSAM Dataset: ACCTDATA.VSAM.KSDS (key: ACCT-ID, 11 digits)

🎯 User Story Development Guide

📝 US Templates

As a CSR, I want to view an account's current balance and credit limit so that I can answer customer balance inquiries.
As a credit officer, I want to update a customer's credit limit so that I can respond to approved limit increase requests.
As a CSR, I want to update an account's active status so that I can freeze or reactivate an account on customer request.
As a CSR, I want to see current cycle credits and debits so that I can explain a customer's statement activity.

✅ Acceptance Criteria Patterns

View Account:
  • Given a valid Account ID, when CAVW is opened, then all account fields displayed: ID, status, current balance, credit limit, cash limit, cycle credits/debits, open/expiry dates
  • Fields must be read-only (ASKIP attribute)
  • PF3 must return to Main Menu without changes
Update Account:
  • Given modified credit limit on CAUP screen, when Enter pressed, then ACCTDATA VSAM updated and "RECORD UPDATED SUCCESSFULLY" shown in GREEN
  • Credit limit must be numeric and positive; reject with error message if not
  • Cash credit limit must be ≤ credit limit; reject otherwise
  • Expiry date must be in YYYY-MM-DD format
  • PF3 must discard changes and return to view screen

🔧 Technical Foundation

🏗️ Key Components

  • COACTVWC: Read account from ACCTDATA VSAM by account ID, populate COACTVW BMS map, display read-only
  • COACTUPC: Read, validate, and rewrite account record; enforces business rule validations before VSAM REWRITE
  • CVACT01Y: Account record copybook (PIC definitions for all account fields)
  • CVACT02Y: Account-card mapping record (links ACCT-ID to CARD-NUM)

🔗 Dependencies

  • authentication: Session must be established (CC00) before CAVW/CAUP access
  • card-management: CARDDATA VSAM references ACCT-ID as foreign key
  • transaction-management: TRANSACT VSAM references account via card number
  • batch-processing: INTCALC and POSTTRAN update ACCTDATA; online updates should not run concurrently with batch
