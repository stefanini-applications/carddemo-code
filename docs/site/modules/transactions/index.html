💸 Transactions

User Story Development Guide — Transaction Browse, View, Add, and Batch Posting

📋 Overview

Module ID: transactions

Purpose: Online browsing, viewing, and adding of credit card transactions; batch posting of daily transactions from sequential DALYTRAN file to TRANSACT VSAM.

Business Context: Transactions are the primary financial activity records. Online transactions are stored in TRANSACT VSAM. Daily batch transactions from DALYTRAN are posted by POSTTRAN (CBTRN02C), which updates account balances and category balances simultaneously.

🎯 User Story Templates

Browse Transactions

As a cardholder, I want to browse my recent transactions so that I can monitor my spending and detect unauthorized charges.
Acceptance Criteria:
  • Transaction CT00 (COTRN00C) lists TRAN-RECORDs from TRANSACT VSAM
  • List shows: TRAN-ID, TRAN-AMT, TRAN-MERCHANT-NAME, TRAN-ORIG-TS, TRAN-TYPE-CD
  • Browsing supports forward/backward paging through records
  • Filter by card number (TRAN-CARD-NUM) using alternate index on TRANSACT

View Transaction Detail

As a cardholder, I want to view the full details of a transaction including merchant information and timestamps so I can verify or dispute a charge.
Acceptance Criteria:
  • Transaction CT01 (COTRN01C) shows all TRAN-RECORD fields
  • Displays: TRAN-ID, TRAN-TYPE-CD with description, TRAN-CAT-CD with description, TRAN-SOURCE, TRAN-DESC, TRAN-AMT, TRAN-MERCHANT-* fields, TRAN-CARD-NUM, TRAN-ORIG-TS, TRAN-PROC-TS
  • Transaction type description looked up from TRANTYPE VSAM
  • Category description looked up from TRANCATG VSAM

Add Transaction

As a teller, I want to manually add a transaction record so that adjustments and payments are captured in the system.
Acceptance Criteria:
  • Transaction CT02 (COTRN02C) provides input form for new TRAN-RECORD
  • Required fields: TRAN-TYPE-CD, TRAN-CAT-CD, TRAN-AMT, TRAN-CARD-NUM, TRAN-DESC, TRAN-MERCHANT-ID
  • TRAN-ID generated by system; TRAN-ORIG-TS set to current timestamp
  • Validates TRAN-CARD-NUM exists in CARDFILE before accepting
  • After submit, new record written to TRANSACT VSAM
  • ACCT-CURR-BAL updated in ACCTFILE for associated account

Post Daily Transactions (Batch)

As a batch operations analyst, I want to run the POSTTRAN job nightly so that daily transactions are applied and account balances are updated end-of-day.
Acceptance Criteria:
  • POSTTRAN (CBTRN02C) reads all records from DALYTRAN sequential file
  • For each transaction: validates card/account, posts to TRANSACT VSAM, updates ACCT-CURR-BAL and cycle accumulators
  • Updates TCATBALF (category balance) for each posted transaction
  • Rejected transactions written to DALYREJS file with error codes
  • Job completes with RC=0 for full success; RC=4 for partial rejects; RC=8 for fatal error

📊 Complexity Guidelines

✅ Simple (1-2 pts)

  • Add transaction type description lookup on list screen
  • Add date range filter to transaction browse (CT00)
  • Format TRAN-AMT with sign indicator (debit/credit)

⚠️ Medium (3-5 pts)

  • Add transaction search by merchant name
  • Transaction dispute flagging (new field in TRAN-RECORD)
  • Running balance display on transaction list

🔴 Complex (5-8 pts)

  • POSTTRAN enhancements: parallel processing by account range
  • Transaction reversal workflow (linked original/reversal records)
  • Fraud detection rules in POSTTRAN (amount thresholds, frequency)

TRAN-RECORD (CVTRA05Y, 350 bytes)

TRAN-ID             PIC X(16)       -- Unique transaction ID (key)
TRAN-TYPE-CD        PIC X(02)       -- Type code (links to TRANTYPE)
TRAN-CAT-CD         PIC 9(04)       -- Category code (links to TRANCATG)
TRAN-SOURCE         PIC X(10)       -- Source system identifier
TRAN-DESC           PIC X(100)      -- Transaction description
TRAN-AMT            PIC S9(09)V99   -- Amount: positive=debit, negative=credit
TRAN-MERCHANT-ID    PIC 9(09)       -- Merchant ID
TRAN-MERCHANT-NAME  PIC X(50)       -- Merchant name
TRAN-MERCHANT-CITY  PIC X(50)       -- Merchant city
TRAN-MERCHANT-ZIP   PIC X(10)       -- Merchant ZIP
TRAN-CARD-NUM       PIC X(16)       -- Card used (AIX key)
TRAN-ORIG-TS        PIC X(26)       -- Original timestamp
TRAN-PROC-TS        PIC X(26)       -- Processing timestamp
