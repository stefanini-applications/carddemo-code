🏦 Accounts

User Story Development Guide — Account View, Update, and Interest Calculation

📋 Overview

Module ID: accounts

Purpose: Provides view and update capabilities for credit card account master data including current balances, credit limits, dates, and the account group ID used for interest rate determination.

Business Context: The account record is the financial core of CardDemo. Every card, transaction, and interest calculation links to an account. Account Group ID links to disclosure groups which define interest rates per transaction type/category.

🎯 User Story Templates

View Account

As a cardholder, I want to view my account details (balance, credit limit, dates) so that I know my current financial position.
Acceptance Criteria:
  • Transaction CAVW displays ACCOUNT-RECORD fields from ACCTFILE for the logged-in user's account
  • Screen shows: ACCT-ID, ACCT-CURR-BAL, ACCT-CREDIT-LIMIT, ACCT-CASH-CREDIT-LIMIT, ACCT-OPEN-DATE, ACCT-EXPIRAION-DATE, ACCT-ACTIVE-STATUS
  • If account not found, display appropriate error message; do not crash
  • Read-only display — no modification allowed on this screen

Update Account

As a cardholder, I want to update my account ZIP code and group information so that my billing details are accurate.
Acceptance Criteria:
  • Transaction CAUP (COACTUPC) allows update of permitted fields
  • ACCT-ADDR-ZIP can be updated by cardholder; limits require admin override
  • After successful update, ACCTFILE VSAM record is written with new values
  • Validation: ZIP code format, field length constraints enforced before write
  • Invalid input displays error on screen without losing entered data

Interest Calculation

As a finance operations analyst, I want to run the monthly interest calculation batch job so that interest charges are correctly applied to all accounts.
Acceptance Criteria:
  • INTCALC (CBACT04C) reads TCATBALF sequentially and DISCGRP for rates
  • Interest calculated per account/type/category using DIS-INT-RATE
  • ACCT-CURR-BAL updated in ACCTFILE after interest posting
  • Cycle credit/debit accumulators (ACCT-CURR-CYC-CREDIT, ACCT-CURR-CYC-DEBIT) reset after calculation
  • Job completes with return code 0 for success; non-zero indicates error requiring investigation

📊 Complexity Guidelines

✅ Simple (1-2 pts)

  • Add new display-only field to COACTVW screen
  • Format balance display with commas/decimals
  • Add credit utilization percentage calculation

⚠️ Medium (3-5 pts)

  • Add credit limit increase request workflow
  • Add account status change (activate/deactivate)
  • Create new batch report for accounts near credit limit

🔴 Complex (5-8 pts)

  • Implement tiered interest rate calculation by balance range
  • Add account history GDG-based tracking across cycles
  • Cross-file validation: account + customer + card consistency check
