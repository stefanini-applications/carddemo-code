🔐 Authentication

User Story Development Guide — CardDemo Signon & Session Management

← Back to Documentation Index

Module ID: authentication

Purpose: Handles user signon and session initialization for the CardDemo CICS application. Validates credentials against the USRSEC VSAM file and routes users to the appropriate menu (regular or admin) based on user type.

Tags: CICS VSAM

🎯 User Story Development Guide

📝 US Templates

  • Login: As a regular user, I want to log in with my user ID and password so that I can access card management functions.
  • Admin Login: As an admin user, I want to log in with admin credentials so that I can manage users and transaction types.
  • Invalid Credentials: As a user, I want to see a clear error message when my credentials are invalid so that I know I need to retry.
  • Exit: As a user, I want to press PF3 to exit the signon screen so that I can close the session.
  • Session State: As a system, I want to carry user identity in the COMMAREA so that all subsequent transactions know the authenticated user.

⚡ Development Acceleration Factors

  • COCOM01Y: Common COMMAREA copybook — user ID, type, and session flags shared across all programs
  • CSUSR01Y: User security record structure — reused in user management module
  • CSDAT01Y / CSMSG01Y / COTTL01Y: Shared date, message, and title copybooks — consistent screen layout across the application
  • COSGN00 (BMS map): Pre-built 3270 signon screen layout

🔧 Technical Foundation

🏗️ Key Components

  • COSGN00C (CICS program) — Signon screen logic; reads USRSEC VSAM file using CICS READ command
  • COSGN00 (BMS mapset) — 3270 terminal layout; fields: USERID, PASSWORD, ERRMSG
  • COMEN01C (CICS program) — Main menu for regular users; routed to after successful signon
  • COADM01C (CICS program) — Admin menu; routed to for admin users (SEC-USR-TYPE = 'A')
  • CSUSR01Y (copybook) — User security data structure
  • COCOM01Y (copybook) — Common COMMAREA with CDEMO-* fields

🔗 Dependencies

  • USRSEC VSAM file: Stores all user records (SEC-USR-ID as KSDS key)
  • RACF: Platform-level security; CICS must be authorized to read USRSEC
  • CICS COMMAREA: 32KB max; carries session state through pseudo-conversational transactions

🌐 CICS Transactions

CC00  → COSGN00C  — Signon screen (entry point for all users)
CM00  → COMEN01C  — Regular user main menu (post-signon for 'R' type users)
CA00  → COADM01C  — Admin menu (post-signon for 'A' type users)

📋 Data Model

SEC-USER-DATA (CSUSR01Y, 80 bytes, USRSEC KSDS):
  SEC-USR-ID      PIC X(08)   — User ID (primary key)
  SEC-USR-FNAME   PIC X(20)   — First name
  SEC-USR-LNAME   PIC X(20)   — Last name
  SEC-USR-PWD     PIC X(08)   — Password (plain text)
  SEC-USR-TYPE    PIC X(01)   — 'R'=Regular, 'A'=Admin
  SEC-USR-FILLER  PIC X(23)

⚙️ Key CICS Commands Used

EXEC CICS READ FILE('USRSEC')
    INTO(SEC-USER-DATA)
    RIDFLD(WS-USER-ID)
    RESP(WS-RESP-CD)
END-EXEC

EXEC CICS RETURN
    TRANSID(WS-TRANID)
    COMMAREA(CARDDEMO-COMMAREA)
END-EXEC

📋 Business Rules

  • BR-AUTH-01: Users must enter a valid User ID (8 chars) and Password (8 chars) to gain access
  • BR-AUTH-02: User type ('R' = Regular, 'A' = Admin) determines menu routing
  • BR-AUTH-03: Invalid credentials display an error on the signon screen; screen is re-presented
  • BR-AUTH-04: PF3 displays "Thank You" message and exits the application
  • BR-AUTH-05: ENTER key triggers credential validation

✅ Acceptance Criteria Examples

  • Successful Login: Given valid user ID and password, when user presses ENTER, then the user is routed to the main menu appropriate for their user type
  • Invalid Login: Given invalid credentials, when user presses ENTER, then the signon screen is redisplayed with an error message, and the user ID field retains the entered value
  • Admin Routing: Given user type = 'A', when signon succeeds, then the admin menu (CA00) is displayed
  • Regular User Routing: Given user type = 'R', when signon succeeds, then the regular menu (CM00) is displayed
  • PF3 Exit: Given the signon screen is displayed, when user presses PF3, then "Thank You" message is shown and session ends
  • Performance: Signon must complete within 3 seconds from Enter key
