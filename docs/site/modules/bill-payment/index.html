💰 Bill Payment

User Story Development Guide — CardDemo Mainframe Application

📋 Overview

Module ID: bill-payment

Purpose: Online bill payment processing — pays the current account balance in full and records the payment as a transaction in the transaction file.

Business Context: Simulates a customer paying their monthly credit card bill. The payment reduces the account balance to zero and creates an offsetting credit transaction.

Transaction: CB00 Program: COBIL00C BMS Map: COBIL00

🎯 User Story Development Guide

📝 US Templates

  • As a regular user, I want to pay my current bill balance online so that I can settle my account
  • As a regular user, I want to see a payment confirmation so that I know my payment was processed
  • As a regular user, I want to see my current balance before paying so that I know the amount being charged

⚡ Development Acceleration Factors

  • COBIL00C: Complete bill payment implementation as reference — shows full-balance payment pattern
  • CVACT01Y: Account record layout used to read and update ACCT-CURR-BAL
  • CVTRA05Y: Payment recorded as new TRAN-RECORD with credit amount
  • Combines account UPDATE (REWRITE to ACCTFILE) with transaction WRITE (to TRANSACT) in same CICS task

�� Complexity Guidelines

  • Simple (1-2 pts): Display current balance from ACCTFILE for confirmation screen
  • Medium (3-5 pts): Full payment processing — read account, create payment transaction, rewrite account balance (CB00)
  • Complex (5-8 pts): Partial payment option, EFT account integration, payment scheduling

⚙️ Business Rules

  • Payment amount = current account balance (ACCT-CURR-BAL) — full balance payment only in base version
  • Payment generates a new transaction record in TRANSACT file with credit amount
  • ACCT-CURR-BAL set to 0.00 after successful payment processing
  • EFT account ID (CUST-EFT-ACCOUNT-ID from CUSTFILE) can be used as funding source reference
  • Account must be ACCT-ACTIVE-STATUS='Y' for payment to proceed
  • If account balance = 0, payment screen should indicate no balance due

✅ Acceptance Criteria Patterns

  • Given an account with outstanding balance, when bill payment is submitted, then account balance is set to 0 and a payment transaction is written to TRANSACT
  • Given a payment confirmation screen, when payment is confirmed, then ACCTFILE is rewritten and success message is displayed
  • Given an account with zero balance, when bill payment screen is opened, then "No amount due" message is displayed
  • Given ACCTFILE REWRITE fails, when payment is attempted, then an error message is shown and no transaction record is created
