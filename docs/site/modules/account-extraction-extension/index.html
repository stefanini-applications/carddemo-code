ЁЯУж Account Extraction Extension

Guide for Developing User Stories

ЁЯУЛ Overview

Purpose: Demonstrates MQ-driven account and date inquiries that read VSAM data without forcing clients into the 3270 UI.

Business Context: Highlights asynchronous MQ patterns and message correlation while exposing VSAM records to downstream services.

ЁЯОп User Story Development Guide

ЁЯУЭ Module-Specific US Templates

  • System date query: As an automation script, I want `CDRD` to return the current date via MQ so that nightly processes stay synchronized.
  • Account inquiry: As an external service, I want `CDRA` to fetch VSAM-backed account details through `CARDDEMO.RESPONSE.QUEUE` without logging into CICS.
  • Extraction resilience: As an operator, I want error handling on MQ listeners so transient queue outages do not leave requests hanging.

тЪб Development Acceleration Factors

  • Message formats: Borrow structures from `app/app-vsam-mq/README.md` such as `DATE-REQUEST`/`ACCT-REQUEST` blocks to keep payloads consistent.
  • MQ samples: Existing definitions under `csd/` show `MQCONN`, `MQQUEUE`, and correlation ID usage that can be reused.
  • BMS fallback: If MQ needs to show data on-screen, reuse the forms from base module to mirror layout while the request is processed asynchronously.

ЁЯУК Complexity Guidelines

  • Simple (1-2 pts): Update the `ACCT-RESPONSE` payload to include one extra field with no MQ logic change.
  • Medium (3-5 pts): Add new correlation ID handling or queue attributes to `CDRA` so multi-threaded clients are supported.
  • Complex (5-8 pts): Build a new extraction transaction that streams VSAM rows into an MQ batch file with chunking support.

тЬЕ Acceptance Criteria Examples

  • `CDRD` sends `DATE-REQUEST`, waits for `DATE-RESPONSE`, and returns `SYSTEM-DATE` on-screen if the MQ reply arrives within a single CICS unit of work.
  • `CDRA` handles missing account numbers by returning a response with `RESPONSE-TYPE = 'ACCT'` and a clear error indicator.
  • MQ connection definitions (`MQCONN`, `MQQUEUE`) remain registered even after a return code <> 0, avoiding orphaned channel handles.

ЁЯФз Technical Foundation

ЁЯПЧя╕П Key Components

  • CICS programs: `CODATE01` (`CDRD`) and `COACCT01` (`CDRA`) that wrap MQ request/response sequences.
  • MQ resources: `CARDDEMO.REQUEST.QUEUE` / `CARDDEMO.RESPONSE.QUEUE`, MQCONN, and correlation IDs described in the README.
  • Message structures: `DATE-REQUEST`, `DATE-RESPONSE`, `ACCT-REQUEST`, `ACCT-RESPONSE` with fixed-length fields.
  • VSAM access: These programs read from `AWS.M2.CARDDEMO.ACCTDATA.PS` using existing copybooks.

ЁЯФЧ Dependencies

  • IBM MQ subsystem with defined queues and channels accessible from CICS.
  • Base CardDemo for the underlying VSAM files and account copybooks.
  • CICS MQ support (the runtime must have MQCONN/MQOPEN privileges for queue access).

ЁЯМР Public Interfaces


CICS:
- CDRD / CODATE01 -> System Date MQ request/response
- CDRA / COACCT01 -> Account inquiry MQ request/response
MQ:
- CARDDEMO.REQUEST.QUEUE -> Receives DATE/ACCT requests
- CARDDEMO.RESPONSE.QUEUE -> Delivers DATE/ACCT responses
