💳 Card Management

User Story Development Guide · CardDemo System

📋 Module Overview

Purpose: List, view, and update credit cards associated with accounts. Cards are identified by 16-digit encrypted card numbers stored in CARDDATA VSAM with an alternate index (CARDAIX) for card-to-account lookups.

Business Context: CSRs need to identify specific cards when customers call, view card details, and perform actions like blocking a card or updating limits. The alternate index (CARDAIX) enables lookups by card number when account ID is not known.

Transaction IDs: CCLI (List) CCDL (Detail) CCUP (Update)

Key Programs: COCRDLIC COCRDSLC COCRDUPC

VSAM Datasets: CARDDATA.VSAM.KSDS, CARDXREF.VSAM.KSDS (CARDAIX AIX)

🎯 User Story Development Guide

📝 US Templates

As a CSR, I want to list all cards for an account so that I can identify which card a customer is calling about.
As a CSR, I want to view card details (masked number, expiry, status) so that I can confirm the card identity without exposing the full number.
As a fraud analyst, I want to set a card's active status to N so that I can immediately block a compromised card.
As a credit officer, I want to update a card's credit limit so that it reflects an approved limit change.

✅ Acceptance Criteria Patterns

Card List Screen (CCLI):
  • Must display up to 15 cards per page with PF7/PF8 pagination
  • Card number must be masked (display only last 4 digits)
  • Enter 'S' in selection field to navigate to card detail (CCDL)
  • PF3 returns to Main Menu
Card Update (CCUP):
  • Card status field accepts only 'Y' (active) or 'N' (inactive)
  • Expiry date must be in YYYY-MM-DD format and not in the past
  • "RECORD UPDATED SUCCESSFULLY" shown in GREEN on successful VSAM REWRITE
  • CARDXREF VSAM must be updated if card-account relationship changes

🔧 Technical Foundation

🏗️ Key Components

  • COCRDLIC: Reads CARDDATA VSAM sequentially for account (STARTBR/READNEXT), populates pageable list
  • COCRDSLC: Direct VSAM READ by card key, maps to COCRDSL BMS display
  • COCRDUPC: Read-validate-REWRITE pattern for CARDDATA VSAM updates
  • CVCARD01Y: Card record layout (card number encrypted, account ID, CVV, embossed name, expiry, status)
  • CARDAIX: Alternate index path — enables lookup by card number returning account ID

🔗 Dependencies

  • account-management: CARD-ACCT-ID must reference a valid account in ACCTDATA VSAM
  • transaction-management: TRAN-CARD-NUM links transactions to cards
  • CARDXREF.VSAM.KSDS: Must be rebuilt (TRANIDX job) after bulk card data loads
