💰 Transaction Management

User Story Development Guide · CardDemo System

📋 Module Overview

Purpose: List, view, and add credit card transactions. Online entry populates TRANSACT VSAM; daily batch (POSTTRAN) is the authoritative posting mechanism for daily file transactions.

Business Context: CSRs need to view transaction history for dispute resolution and credit analysts need to add manual transactions (fees, credits, adjustments). Transaction data is the core audit trail for all card activity.

Transaction IDs: CT00 (List) CT01 (View) CT02 (Add)

Key Programs: COTRN00C COTRN01C COTRN02C

VSAM Dataset: TRANSACT.VSAM.KSDS (key: TRAN-ID, 16 chars)

🎯 User Story Development Guide

📝 US Templates

As a CSR, I want to view all transactions for a card so that I can help a customer dispute a charge.
As a CSR, I want to page through transaction history (PF7/PF8) so that I can find a specific transaction from a prior month.
As a credit analyst, I want to add a manual transaction (fee or credit) so that I can apply a one-time adjustment to an account.
As a CSR, I want to view full transaction details (merchant, amount, timestamps) so that I can provide complete information to a customer.

✅ Acceptance Criteria Patterns

Transaction List (CT00):
  • Must display 10–15 transactions per page with PF7/PF8 pagination
  • Shows transaction ID, type, amount, merchant name, and original timestamp
  • Enter 'S' in selection field to view detail (CT01)
  • At first page, PF7 must show "NO MORE RECORDS" message; at last page, PF8 similar
Add Transaction (CT02):
  • Transaction type code must exist in TRANTYPE file; reject with error if not
  • Amount field must be numeric (S9(10)V99); reject non-numeric input
  • Transaction ID generated by program (must be unique in TRANSACT VSAM)
  • Processing timestamp set to current system timestamp on write
  • Duplicate VSAM key must show "DUPLICATE TRANSACTION ID" error
  • "TRANSACTION ADDED SUCCESSFULLY" shown in GREEN on success

🔧 Technical Foundation

🏗️ Key Components

  • COTRN00C: Sequential VSAM STARTBR/READNEXT for paging; stores first-of-page key in COMMAREA for PF7 backward navigation
  • COTRN01C: Direct VSAM READ by TRAN-ID key
  • COTRN02C: VSAM WRITE with duplicate key checking; validates TRAN-TYPE-CD against TRANTYPE dataset
  • CVTRA05Y: Transaction record copybook — 16-char ID, type code, category, amount (COMP-3), merchant fields, card number, dual timestamps

🔗 Dependencies

  • card-management: TRAN-CARD-NUM must reference a valid card
  • transaction-types (optional): If DB2 module installed, type codes validated against DB2 table; otherwise validated against TRANTYPE flat file
  • batch-processing (POSTTRAN): Batch daily transactions from DALYTRAN file; online CT02 entry writes directly to TRANSACT VSAM
