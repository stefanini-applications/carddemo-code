💳 Credit Cards

User Story Development Guide — Card List, View, and Update

📋 Overview

Module ID: credit-cards

Purpose: Manages credit card records linked to accounts — listing, viewing details, and updating card attributes including status, embossed name, and expiration date.

Business Context: Each account may have one or more cards. The CARDXREF file provides the three-way linkage between card number, customer ID, and account ID, enabling lookups from any of the three keys.

🎯 User Story Templates

List Credit Cards

As a cardholder, I want to see a list of all credit cards associated with my account so that I can manage my active and inactive cards.
Acceptance Criteria:
  • Transaction CCLI (COCRDLIC) displays all CARD-RECORDs for the current account
  • List shows: CARD-NUM (masked), CARD-ACTIVE-STATUS, CARD-EMBOSSED-NAME, CARD-EXPIRAION-DATE
  • If no cards found for account, display informational message
  • User can select a card from the list to proceed to view (CCDL) or update (CCUP)

View Card Details

As a cardholder, I want to view detailed information about a specific credit card so that I can verify the card's status and expiration.
Acceptance Criteria:
  • Transaction CCDL (COCRDSLC) shows all CARD-RECORD fields for selected card
  • CVV (CARD-CVV-CD) is displayed only in authorized contexts (not masked in base app)
  • CARD-ACCT-ID confirms the account link is displayed
  • Read-only — no updates from this screen

Update Card

As a customer service agent, I want to update a credit card's status and embossed name so that the card record reflects current information.
Acceptance Criteria:
  • Transaction CCUP (COCRDUPC) allows update of CARD-ACTIVE-STATUS, CARD-EMBOSSED-NAME, CARD-EXPIRAION-DATE
  • After submit, CARDFILE VSAM record is rewritten
  • CARD-NUM (key) and CARD-ACCT-ID cannot be modified through this screen
  • Validation: active status must be valid value; expiration date format YYYY-MM-DD
  • Success confirmation displayed; error redisplays with original data

📊 Complexity Guidelines

✅ Simple (1-2 pts)

  • Mask all but last 4 digits of CARD-NUM on list screen
  • Add card expiration warning indicator (within 30 days)
  • Sort card list by active status then card number

⚠️ Medium (3-5 pts)

  • Add card replacement workflow (deactivate old, create new)
  • Add card lost/stolen flag with immediate deactivation
  • CARDXREF lookup — find account from card number

🔴 Complex (5-8 pts)

  • Batch card expiration notification report (cards expiring in 60 days)
  • New card issuance workflow with CARDXREF creation
  • Card number generation algorithm with Luhn check
