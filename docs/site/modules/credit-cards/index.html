💳 Credit Cards

User Story Development Guide — CardDemo Mainframe Application

📋 Overview

Module ID: credit-cards

Purpose: List, view, and update credit card records. Cards are linked to accounts via the CARDXREF cross-reference VSAM file.

Business Context: Each account can have multiple cards. The cross-reference (CVACT03Y) maps card number → customer ID + account ID, enabling navigation from card to account and vice versa.

CCLI: COCRDLIC (list) CCDL: COCRDSLC (view) CCUP: COCRDUPC (update) VSAM: CARDFILE + XREFFILE

🎯 User Story Development Guide

📝 US Templates

  • As a regular user, I want to list all cards on my account so that I can see which cards are active
  • As a regular user, I want to view card details (number, expiry, CVV, status) so that I can verify my card information
  • As a regular user, I want to update my card active status so that I can activate or deactivate a card
  • As a regular user, I want to see the embossed name on my card so that I can confirm it is correct

⚡ Development Acceleration Factors

  • CVACT02Y copybook: Complete 150-byte CARD-RECORD layout — card number, account ID, CVV, name, expiry, status
  • CVACT03Y copybook: 50-byte CARD-XREF-RECORD — maps card to customer and account (XREF-CARD-NUM → XREF-CUST-ID + XREF-ACCT-ID)
  • AIX (Alternate Index): XREFFILE supports lookup by account ID as well as card number for list operations

📊 Complexity Guidelines

  • Simple (1-2 pts): Card detail view from single VSAM read (CCDL)
  • Medium (3-5 pts): Card list via XREFFILE AIX browse + pagination (CCLI); card status update (CCUP)
  • Complex (5-8 pts): Cross-module — card deactivation triggering account available credit recalculation

📋 Card Data Model

01  CARD-RECORD.                   -- 150 bytes
    05  CARD-NUM              PIC X(16). -- Primary key
    05  CARD-ACCT-ID          PIC 9(11).
    05  CARD-CVV-CD           PIC 9(03).
    05  CARD-EMBOSSED-NAME    PIC X(50).
    05  CARD-EXPIRAION-DATE   PIC X(10). -- YYYY-MM-DD (typo preserved)
    05  CARD-ACTIVE-STATUS    PIC X(01). -- 'Y'=active
    05  FILLER                PIC X(59).

01  CARD-XREF-RECORD.             -- 50 bytes
    05  XREF-CARD-NUM         PIC X(16). -- Primary key
    05  XREF-CUST-ID          PIC 9(09).
    05  XREF-ACCT-ID          PIC 9(11).
    05  FILLER                PIC X(14).

⚙️ Business Rules

  • Card list filtered by account ID using XREFFILE AIX browse
  • CARD-ACTIVE-STATUS controls card usability; 'Y'=active, 'N'=inactive
  • Cross-reference (XREFFILE) must be consistent with CARDFILE — updated together
  • CVV stored as 3-digit numeric in source data
  • CARD-EXPIRAION-DATE field name has intentional typo — preserved for compatibility

✅ Acceptance Criteria Patterns

  • Given a valid account ID, when credit card list is requested, then all cards for that account are displayed with card number and status
  • Given a valid card number, when card view is requested, then full card details are displayed
  • Given updated card active status, when card update is submitted, then CARDFILE is rewritten and success message displayed
  • Given no cards on file for an account, when card list is requested, then "No cards found" message is displayed
