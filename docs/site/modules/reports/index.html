📊 Reports

User Story Development Guide · CardDemo System

📋 Module Overview

Purpose: Display aggregated transaction statistics and summaries by category, sourced from the transaction category balance file (TCATBALF).

Business Context: Managers and compliance officers need summary views of transaction activity. The report screen provides read-only aggregated views — not raw transaction listings. Category totals are pre-computed by batch processing.

Transaction ID: CR00

Key Programs: CORPT00C

Key Data Sources: TCATBALF.PS (category balances), TRANCATG.PS (category descriptions)

🎯 User Story Development Guide

📝 US Templates

As a branch manager, I want to view transaction totals by category so that I can monitor daily spending patterns across the portfolio.
As a compliance officer, I want to see aggregated transaction category counts so that I can verify daily processing totals match expected volumes.
As a data analyst, I want category descriptions displayed alongside codes so that I can interpret report data without looking up codes separately.

✅ Acceptance Criteria Patterns

Report Display (CR00):
  • Category codes and descriptions displayed side by side from TRANCATG file
  • Transaction counts and amounts formatted (e.g., +99,999,999.99) from TCATBALF
  • Report data reflects last completed batch cycle (POSTTRAN)
  • Screen is read-only (all fields ASKIP) — no data entry
  • PF3 returns to Main Menu

🔧 Technical Foundation

🏗️ Key Components

  • CORPT00C: Reads TCATBALF sequential file, joins with TRANCATG for descriptions, formats amounts, populates CORPT00 BMS map
  • TCATBALF.PS: Category balance flat file — updated by batch POSTTRAN; contains count and amount per category code
  • TRANCATG.PS: Transaction category master — static lookup for category descriptions

🔗 Dependencies

  • batch-processing (POSTTRAN): TCATBALF is populated by nightly batch; report data is batch-cycle current, not real-time
  • transaction-management: Source transactions come from TRANSACT VSAM, summarized into TCATBALF by batch
