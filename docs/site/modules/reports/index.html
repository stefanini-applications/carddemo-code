📊 Reports

User Story Development Guide — CardDemo Mainframe Application

📋 Overview

Module ID: reports

Purpose: Online initiation of transaction reports (CORPT00C via CR00) and batch production of transaction detail reports (CBTRN03C) and account statements in plain text and HTML (CBSTM03A/CBSTM03B).

Business Context: Report generation bridges the online and batch worlds — users request reports online via CICS, which submits JCL to an extra-partition TDQ for batch execution. Statements are a key customer-facing deliverable.

CR00: CORPT00C (CICS initiation) TRANREPT: CBTRN03C (batch report) CREASTMT: CBSTM03A+B (batch statement) TDQ: Extra-partition

🎯 User Story Development Guide

📝 US Templates

  • As a regular user, I want to generate a transaction report so that I can download a history of all transactions for a date range
  • As a regular user, I want to receive a monthly account statement so that I can review my billing cycle activity
  • As a system operator, I want batch transaction reports to run automatically so that report output is available each morning
  • As a regular user, I want my statement available in both text and HTML formats so that I can read it in different contexts

⚡ Development Acceleration Factors

  • TDQ pattern: CORPT00C writes JCL to extra-partition TDQ which triggers batch execution — established pattern for CICS→batch bridge
  • CBSTM03A: Demonstrates multiple COBOL techniques (control block addressing, ALTER/GOTO, COMP/COMP-3 variables, 2-D arrays, subroutine calls)
  • CBTRN03C: Reads TRANSACT + XREFFILE + TRANTYPE + TRANCATG — reference for multi-file sequential processing
  • HTML output: CBSTM03A generates both plain text (STMTFILE) and HTML (HTMLFILE) simultaneously

📊 Complexity Guidelines

  • Simple (1-2 pts): Report request screen with date range input (CORPT00C screen)
  • Medium (3-5 pts): TDQ submission of JCL, status feedback to user (CR00)
  • Complex (5-8 pts): Batch report with multi-file cross-referencing, HTML/text output, pagination (CBTRN03C, CBSTM03A)
  • Very Complex (8-13 pts): Real-time report status tracking, PDF generation (TXT2PDF JCL utility available)

🔧 Technical Foundation

⚙️ Business Rules

  • CORPT00C accepts date range (from/to) and account ID for report scope
  • JCL submitted to extra-partition TDQ triggers asynchronous batch execution
  • Report output written to SYSOUT/report datasets — not displayed online
  • CBTRN03C reads TRANSACT sequentially and cross-references XREFFILE, TRANTYPE, TRANCATG for each record
  • CBSTM03A reads TRANSACT + ACCTFILE + CUSTFILE to produce customer-level statements
  • HTML statement output uses inline HTML generation from COBOL (no external template engine)
  • TXT2PDF JCL utility (TXT2PDF1.JCL) available for converting text output to PDF

✅ Acceptance Criteria Patterns

  • Given a valid date range, when report request is submitted via CR00, then JCL is written to TDQ and a submission confirmation is displayed
  • Given the TRANREPT batch job runs, when CBTRN03C completes, then all transactions in the date range are included in the report output with type and category descriptions
  • Given the CREASTMT batch job runs, when CBSTM03A completes, then both plain text and HTML statement files are produced for each account with activity
  • Given no transactions in the date range, when report is generated, then report contains "No transactions found" message with report header
