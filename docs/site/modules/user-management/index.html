👥 User Management

User Story Development Guide · CardDemo System

📋 Module Overview

Purpose: Admin-only CRUD operations for system users — add, update, and delete bank operator accounts with role-based type assignment (Admin vs User).

Business Context: System admins provision, manage, and revoke operator access. User type determines which screens are accessible: Admin users can access user management and optional admin screens; regular Users access only core card/account functions.

Transaction IDs: CU00 (List) CU01 (Add) CU02 (Update) CU03 (Delete)

Key Programs: COUSR00C COUSR01C COUSR02C COUSR03C

VSAM Dataset: USRSEC.VSAM.KSDS (key: SEC-USR-ID, 8 chars)

⚠️ Admin-only access required. User type must be 'A' (Admin) to access CU00–CU03 screens.

🎯 User Story Development Guide

📝 US Templates

As a system admin, I want to add a new user so that a new bank employee can log in and access the system.
As a security admin, I want to update a user's type to Admin so that they can manage other users and access admin functions.
As a security admin, I want to delete a terminated employee's account so that their access is revoked immediately.
As a system admin, I want to list all system users so that I can audit who has access to the system.
As a security admin, I want to reset a user's password so that I can restore access for a locked-out operator.

✅ Acceptance Criteria Patterns

Add User (CU01):
  • User ID must be unique in USRSEC VSAM; duplicate shows "USER ID ALREADY EXISTS"
  • User type field accepts only 'A' (Admin) or 'U' (User)
  • Password field stored encrypted; not displayed after creation
  • All name fields (first, last) required — blank shows field-specific error
  • "USER ADDED SUCCESSFULLY" in GREEN on successful VSAM WRITE
Delete User (CU03):
  • Must display user details and require confirmation before VSAM DELETE
  • Cannot delete the last Admin user; show "CANNOT DELETE LAST ADMIN" error
  • Cannot delete currently logged-in user's own record
  • PF3 cancels deletion and returns to user list

🔧 Technical Foundation

🏗️ Key Components

  • COUSR00C: Sequential VSAM STARTBR/READNEXT for paging through USRSEC user list
  • COUSR01C: VSAM WRITE with duplicate key check; applies password encryption before write
  • COUSR02C: VSAM READ then REWRITE; preserves existing password if new password field blank
  • COUSR03C: VSAM READ (confirm display) then DELETE; includes last-admin guard
  • CSUSR01Y: User security record copybook (ID 8, fname 20, lname 20, password 8 encrypted, type 1)

🔗 Dependencies

  • authentication: USRSEC VSAM shared with COSGN00C; changes take effect on next login
  • COADM01C (Admin Menu): Routes admin users to CU00 list; regular users do not see user management option
  • DUSRSECJ batch job: Initial population of USRSEC VSAM; must run before first use
