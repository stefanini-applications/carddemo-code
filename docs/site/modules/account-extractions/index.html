📡 Account Extractions Module

Guide for MQ-driven extractions that demonstrate asynchronous VSAM integration.

📋 Overview

Purpose: Offer MQ request/response examples (system date via `CDRD`, account data via `CDRA`) so external systems can fetch mainframe state asynchronously.

Business Context: Teams migrating services can rely on these stories to showcase data extraction and correlation patterns (no modern UI) while consuming VSAM sources.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • System Date MQ Call: As a downstream scheduler, I want `CDRD` to return the z/OS system date so cross-system jobs stay in sync.
  • Account Payload: As an integration partner, I want `CDRA` to return the full `ACCOUNT-DATA` payload so I can populate a reporting service.
  • Error Recovery: As a support analyst, I want MQ correlation IDs logged when `CDRA` fails so I can retry with the right message.

⚡ Development Acceleration Factors

  • Message formats: `app/app-vsam-mq/README.md` documents `DATE-REQUEST-MSG`, `DATE-RESPONSE-MSG`, `ACCT-REQUEST-MSG`, and `ACCT-RESPONSE-MSG` layouts.
  • CICS/MQ definitions: Resources under `app/app-vsam-mq/csd` show how to define MQCONN, MQQUEUE, and transactions for `CDRD`/`CDRA`.
  • VSAM sources: Programs read account data from `app/data/ACCTDATA`, reusing copybook structures already consumed by `COACTVW`.

📊 Complexity Guidelines

  • Simple (1-2 pts): Adjust the BMS message line in `CDRA` to surface MQ correlation IDs and status.
  • Medium (3-5 pts): Extend the `ACCT-RESPONSE-MSG` with a new `ACCOUNT-STATUS` field and update the MQ handler to write it.
  • Complex (5-8 pts): Add a new MQ queue and listener program, ensuring the MQCONN definitions, CICS programs, and TTLs are coordinated.

🔧 Technical Foundation

🏗️ Key Components

  • `CODATE01` exposes `CDRD` for system-date MQ requests and responses.
  • `COACCT01` implements `CDRA`, reading VSAM account payloads and posting the 300-byte `ACCOUNT-DATA` in `ACCT-RESPONSE-MSG`.
  • MQ resources and definitions exist in `app/app-vsam-mq/csd`, binding `CARDDEMO.REQUEST.QUEUE` and `CARDDEMO.RESPONSE.QUEUE` to CICS programs.
  • VSAM datasets (`app/data/ACCTDATA`) align with `CVCUS01Y`/`CVACT01Y` copybooks, ensuring the same structural expectations as interactive screens.

🔗 Dependencies

  • IBM MQ for request/response (needs MQCONN, MQQUEUE, MQMD headers).
  • CICS region with MQ support (TSO, MQ runtime).
  • VSAM datasets for account and customer master data.

🗂️ Public Interfaces

CDRD - System date MQ request (request type DATE, response includes SYSTEM-DATE)
CDRA - Account data MQ request (ACCOUNT-DATA payload from VSAM)
CARDDEMO.REQUEST.QUEUE / .RESPONSE.QUEUE - MQ queues defined in app/app-vsam-mq/csd

✅ Acceptance Criteria Examples

  • Every MQ response echoes the originating `REQUEST-ID` so clients correlate results.
  • `CDRA` only returns account data when the requesting account number exists in VSAM; otherwise an MQ error is raised.
  • MQCONN and MQQUEUE definitions match the names used by partner docs to avoid mismatched queue managers.
