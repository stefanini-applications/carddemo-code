⚙️ Batch Processing

User Story Development Guide — Nightly Batch Suite, File Loading, and Interest Calculation

📋 Overview

Module ID: batch-processing

Purpose: Encompasses all batch JCL jobs and COBOL programs responsible for bulk data loading, transaction posting, interest calculation, statement generation, backup management, and VSAM maintenance.

Business Context: Batch processing runs nightly outside business hours (CLOSEFIL → processing → OPENFIL) to ensure VSAM files are not held open by CICS during bulk updates. The full batch sequence refreshes all data, posts transactions, calculates interest, and produces reports.

🎯 User Story Templates

Full Batch Run

As a batch operations manager, I want to run the nightly batch sequence so that all account balances, transaction files, and interest calculations are updated for the new business day.
Acceptance Criteria:
  • CLOSEFIL executes first, closing all CICS-managed VSAM files (RC=0)
  • POSTTRAN (CBTRN02C) posts all DALYTRAN records to TRANSACT and updates ACCTFILE (RC≤4)
  • INTCALC (CBACT04C) calculates and applies interest to all active accounts (RC=0)
  • COMBTRAN merges daily and system transactions (RC=0)
  • CREASTMT generates statements for all accounts (RC=0)
  • OPENFIL executes last, making VSAM files available to CICS (RC=0)
  • Any step with RC>8 halts the chain and alerts operations

Transaction Posting

As a batch developer, I want to understand the POSTTRAN job so I can extend it with new validation or output logic.
Acceptance Criteria:
  • CBTRN02C reads DALYTRAN sequentially record by record
  • For each record: validates TRAN-CARD-NUM via CARDFILE, derives account via CARDXREF, posts to TRANSACT VSAM
  • ACCT-CURR-BAL, ACCT-CURR-CYC-DEBIT/CREDIT updated atomically
  • TCATBALF (category balance) updated for the account/type/category combination
  • Invalid records written to DALYREJS with error code

Interest Calculation

As a finance analyst, I want to verify the interest calculation logic so that monthly charges are accurate and auditable.
Acceptance Criteria:
  • CBACT04C reads TCATBALF (TRAN-CAT-BAL-RECORD) sequentially
  • For each balance: looks up DISCGRP using ACCT-GROUP-ID + TYPE-CD + CAT-CD
  • Interest = TRAN-CAT-BAL × DIS-INT-RATE
  • Interest amount added to ACCT-CURR-BAL in ACCTFILE
  • Cycle accumulators (ACCT-CURR-CYC-DEBIT, ACCT-CURR-CYC-CREDIT) reset to zero after calculation

Environment Refresh

As a test environment administrator, I want to reload all VSAM files from sample data so that the test environment is in a known good state.
Acceptance Criteria:
  • Execute setup jobs in sequence: DEFGDGB → DUSRSECJ → ACCTFILE → CARDFILE → CUSTFILE → XREFFILE → TRANFILE → DISCGRP → TCATBALF → TRANCATG → TRANTYPE → OPENFIL
  • Each job uses IDCAMS REPRO to load from sample EBCDIC data files
  • All RC=0 after execution
  • CICS application accessible via CC00 transaction after OPENFIL

📊 Complexity Guidelines

✅ Simple (1-2 pts)

  • Add RC check and conditional step in existing JCL
  • Add NOTIFY parameter to batch job for email alert
  • Modify DALYREJS report format

⚠️ Medium (3-5 pts)

  • Add new COBOL batch program reading existing VSAM files
  • Create new VSAM KSDS dataset and JCL for new entity
  • Parameterize batch job with symbolic variables

🔴 Complex (5-8 pts)

  • POSTTRAN parallel processing using SORT partitioning
  • GDG-based generation management for transaction archives
  • Exception report with cross-file validation (account + card + transaction)

Full Batch Job Sequence

1.  CLOSEFIL   (IEFBR14)    Close CICS VSAM files
2.  ACCTFILE   (IDCAMS)     Refresh account master
3.  CARDFILE   (IDCAMS)     Refresh card master
4.  XREFFILE   (IDCAMS)     Load card cross-reference
5.  CUSTFILE   (IDCAMS)     Load customer master
6.  TRANBKP    (IDCAMS)     Backup transaction master
7.  TRANCATG   (IDCAMS)     Load transaction categories
8.  TRANTYPE   (IDCAMS)     Load transaction types
9.  DISCGRP    (IDCAMS)     Load disclosure groups
10. TCATBALF   (IDCAMS)     Load category balance file
11. DUSRSECJ   (IEBGENER)   Load user security file
12. POSTTRAN   (CBTRN02C)   Post daily transactions
13. INTCALC    (CBACT04C)   Calculate interest
14. COMBTRAN   (SORT)       Merge transactions
15. CREASTMT   (CBSTM03A)   Generate statements
16. TRANIDX    (IDCAMS)     Rebuild TRANSACT alternate index
17. OPENFIL    (IEFBR14)    Re-open CICS VSAM files
