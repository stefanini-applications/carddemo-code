⚙️ Batch Processing

User Story Development Guide · CardDemo System

📋 Module Overview

Purpose: Nightly automated processing cycle: post daily transactions, calculate interest, generate customer statements, and maintain data files. The authoritative mechanism for daily transaction posting.

Business Context: Every night, the batch cycle processes all transactions that came in through card POS systems, updates account balances, charges interest, and produces statements. This is the backbone of the card processing system — online CICS screens provide query/entry; batch provides the authoritative daily reconciliation.

Key Jobs: POSTTRAN INTCALC CREASTMT COMBTRAN

Key Programs: CBTRN02C CBSTM01B CBACT01C

🔄 Nightly Batch Flow

1. OPENFILE → Open VSAM datasets for batch access
2. COMBTRAN → Sort + merge DALYTRAN.PS input
3. POSTTRAN → Post transactions: TRANSACT VSAM + TCATBALF update → DALYREJS(+1) rejects
4. INTCALC → Calculate interest on balances → update ACCTDATA VSAM
5. CREASTMT → Generate statements from TRANSACT VSAM + TCATBALF
6. CLOSEFIL → Close VSAM datasets

🎯 User Story Development Guide

📝 US Templates

As a batch operator, I want POSTTRAN to run automatically each night so that transactions are posted before business hours begin.
As a compliance officer, I want rejected transactions written to DALYREJS with reason codes so that I can investigate and resubmit them.
As an IT operations engineer, I want OPENFILE and CLOSEFIL jobs so that VSAM datasets are properly managed and not left open overnight.
As a data engineer, I want COMBTRAN to sort the daily transaction file so that POSTTRAN receives records in the correct order for VSAM key sequence.
As a customer, I want monthly statements generated by CREASTMT so that I receive an accurate summary of my card activity.

✅ Acceptance Criteria Patterns

POSTTRAN Job:
  • Job completes RC=0 (no ABENDs, no errors)
  • All records from DALYTRAN.PS processed: count in SYSOUT matches input record count
  • Records written to TRANSACT VSAM = input count minus rejected count
  • Rejected records written to DALYREJS(+1) with rejection reason code and original record
  • TCATBALF.PS updated with accumulated category totals
  • Duplicate transaction IDs → rejection code 'DUP', invalid accounts → 'INV', over-limit → 'OVL'
INTCALC Job:
  • Interest applied to all accounts with outstanding balance > 0
  • Account ACCT-CURR-BAL updated in ACCTDATA VSAM
  • Interest transaction written to TRANSACT VSAM with interest type code
  • No duplicate interest charges for same cycle period

🔧 Technical Foundation

📁 Key Programs & JCL

  • app/cbl/CBTRN02C.cbl — Core transaction poster (POSTTRAN step)
  • app/cbl/CBTRN01C.cbl — Pre-processor / transaction validator
  • app/cbl/CBTRN03C.cbl — Transaction detail processor
  • app/cbl/CBSTM01B.cbl / CBSTM01C.cbl — Statement generation
  • app/cbl/CBACT01C.cbl — Account batch processor (interest)
  • app/jcl/POSTTRAN.jcl — Daily transaction posting job
  • app/jcl/INTCALC.jcl — Interest calculation job
  • app/jcl/CREASTMT.jcl — Statement generation job
  • app/jcl/COMBTRAN.jcl — Sort/combine transaction file
  • app/asm/MVSWAIT.asm — Assembler timer utility for batch delays

🔗 Dependencies

  • transaction-management: TRANSACT VSAM — shared with online CT00/CT01/CT02; coordinate access windows
  • account-management: ACCTDATA VSAM — updated by INTCALC; coordinate with online CAUP
  • VSAM dataset lifecycle: OPENFILE job must succeed before any batch program accesses VSAM; CLOSEFIL must run after
